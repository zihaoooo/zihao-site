A flat meadow. Two parametric rules encode two landscape conditions — a mown path, and water-collecting bowls. The same field, two rules, two spatial readings.
This workflow introduces the core logic of parametric design: write the rule first, then build it in nodes.
Week 9 — Mowing Path
A Pseudocode First On paper — no computers
Before opening Grasshopper, write this rule on paper:
FOR EACH plant in meadow:
  IF distance to path < mowing width
    THEN height = mowed (short)
  ELSE height = unmowed (tall, random)
Identify: what is the input, what is the condition, what are the two outputs.
01 Set Up the Meadow Rhino GH
In Rhino, draw a rectangle for the site boundary — e.g. 100 × 150 ft. In Grasshopper:
  • Populate 2D → the rectangle as region
  • Count slider → start at 200
  • Seed slider → controls the random distribution
02 Random Heights (Unmowed) GH
Give every plant a random unmowed height:
  • Random → domain 2.0 to 6.0 (grass height range)
  • Line SDL → start = Populate 2D points, direction = unit Z, length = Random
You now have a meadow — a field of vertical lines of varying height.
03 Draw + Measure the Path Rhino GH
In Rhino, draw a curve across the site — polyline or arc. Reference it into GH with a Curve parameter, then measure how far each plant sits from it:
  • Curve Closest Point → Populate 2D points + path curve
  • Distance → original points to their closest points on the curve
04 Apply the Rule — Dispatch GH
Split the meadow into mown and unmown by the threshold:
  • Larger Than → A = distance, B = mowing width slider (start at 8.0)
  • Dispatch → data = Line SDL lengths, pattern = Larger Than
  • List A (beyond threshold) → keep the unmowed random heights
  • List B (within threshold) → replace with a short fixed height slider (0.5)
  • Rebuild a Line SDL for each branch
Toggle the mowing width. The mown strip widens and narrows. Move the path curve in Rhino — the mown zone follows live. End of Week 9.
Week 10 — Water Bowls
B Pseudocode First On paper — no computers 
FOR EACH plant in meadow:
  height = remap(distance to nearest bowl, 0 → max, tall → short)
Close to a bowl = tall (wet, lush). Far from a bowl = short (dry).
05 Place Bowl Attractors Rhino GH
In Rhino, place 2–4 point objects on the site — these are the bowl locations. Reference them into GH with a Point parameter. Then find the nearest bowl for each plant:
  • Pull Point → input points = Populate 2D points, geometry = bowl attractors
  • Distance → each plant to its pulled (nearest) bowl point
06 Remap to Height GH
Turn distance into a continuous height gradient:
  • Bounds → on the distance list to get the real min/max
  • Remap Numbers → values = distances, source = Bounds, target = 1.0 (close, tall) to 0.2 (far, short)
Flip the target range so close = tall. This is the gradient — proximity drives lushness.
07 Construct the Ellipsoid GH
Draw each plant as a vertically stretched sphere:
  • Sphere → center = Populate 2D points, radius = remapped height
  • Scale NU → geometry = sphere, X = 1.0, Y = 1.0, Z = remapped height
  • Add a Z Scale slider to tune the vertical exaggeration
Taller and rounder near bowls; short and disc-like further away. One value, two readings.
08 Preview + Adjust Rhino
  • Move a bowl point in Rhino — the tall cluster shifts live
  • Add or remove bowl points — each makes a new zone of lush growth
  • Adjust the Remap target domain to change the height contrast
Key Concepts
Populate 2D vs. grid — Populate 2D distributes points organically within a boundary. The Seed slider changes the distribution without changing the count. It reads as a natural meadow, not a tile pattern.
Dispatch vs. Remap — Dispatch encodes a binary rule (mow / no mow). Remap encodes a continuous gradient (tall near water, short away). Most landscape conditions are gradients; the binary is the readable edge.
Pull Point — works like Closest Point: for each input point, finds the nearest geometry in a set and returns its location. Used here to measure proximity to the nearest bowl across multiple attractors.
Ellipsoid from Scale NU — a sphere scaled non-uniformly in Z. The same remap value drives both radius and Z scale, so plants near bowls are taller and rounder; plants far away are short and flat.

Tips
  • Write the rule on paper before touching a node. The pseudocode is the design.
  • Dispatch is the readable edge; Remap is the gradient. Know which one the condition wants.
  • Set hints and watch the preview live — drag a slider and read the change before moving on.
  • Label your slider values in the screenshots. The deliverable is the relationship, not the picture.
